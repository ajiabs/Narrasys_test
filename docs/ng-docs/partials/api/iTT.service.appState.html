appState
service in module iTT

Description

POJO designed to store application state during app runtime. All values are initiazed as boolean and set to false. Values are then updated in place by injecting the appState into the relevant destination and overwriting the initial values

Dependencies

  • $interval
  • config

Properties

  • user

    whatever authSvc gets back from getAccessToken

  • episodeId

    ID of current episode (Initialized as bool set to false).

  • episodeSegmentId

    ID of current episode segment (only relevant in narratives)

  • narrativeId

    also only relevant in narratives

  • timelineId

    ID of timeline

  • isFramed

    are we inside an iframe? Don't use !== because IE8 gets it wrong

  • isTouchDevice

    Determines whether or not client is touch enabled via user agent

  • isIphone

    iPhone has weird video handling, see timelineSvc

  • windowWidth

    width of content pane, updated every 50ms

  • windowHeight

    height of content pane, updated every 50ms

  • viewMode

    Sets view mode based upon window width

  • producerEditLayer

    a bit hacky, this. Only has an effect in producer in discover mode; 0 is default, -1 is background layers only, 1 is foreground layers only

  • time

    current playhead position (in seconds) relative to timeline NOT TO EPISODE!

  • bufferedPercent

    portion of video that has been buffered (as pct instead of time because that's how youtube reports it, and that's what we end up displaying anyway)

  • timeMultiplier

    sets player speed (0.5 = half speed; 2=double;etc)

  • duration

    duration of timeline (in seconds)

  • timelineState

    "playing", "paused", or "buffering" (set by timelineSvc). Future = "locked" (by stop question or etc)

  • hasBeenPlayed

    set to true after first time the video plays (used so we can interrupt that first play with a helpful help)

  • volume

    Audio for main video

  • muted

    audio for main video

  • hideCaptions

    visibility of "closed captions" in watch mode

  • show

    Object with SearchPanel and NavPanel properties which are booleans

  • videoControlsActive

    whether bottom toolbar is visible

  • videoControlsLocked

    force bottom toolbar to stay in its current visible/hidden state

  • itemDetail

    Put item data here to show it as a modal overlay

  • autoscroll

    scroll window to make current items visible (in relevant modes)

  • autoscrollBlocked

    User has disabled autoscroll

  • product

    "player", "sxs", or "producer"

  • productLoadedAs

    same as product but only set on initial load, this lets producer toggle back to player preview temporarily

  • editEvent

    Scene or item currently being edited by user.

  • editEpisode

    Episode currently being edited by user. yes I did kind of paint myself into a corner here

  • lang

    set to false so the episode default knows when to override it
